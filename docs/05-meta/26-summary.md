* **Admin Portal Overview**

  * **Purpose**

    ```
    -   Internal web application for managing:
        -   Users
        -   Gifts
        -   Templates
        -   Payments and MoMo logs
        -   Notifications (SMS / WhatsApp / Email / Push)
        -   USSD activity
        -   AI usage
        -   System configuration and security
    ```

  * **Access**

    ```
    -   Hosted on a separate domain or subdomain
        -   Example: admin.giftlink.com

    -   Protected with:
        -   Email + strong password
        -   Optional 2FA (SMS or Authenticator app)
    ```

  * **Roles & Permissions**

    ```
    At minimum, the system should support these roles:

    1.  Super Admin
        -   Full access to everything
        -   Can create, edit, and deactivate other admin accounts

    2.  Support Admin
        -   View users, gifts, and notifications
        -   Resend notifications
        -   Adjust delivery status (where business rules allow)
        -   Cannot change pricing, fees, or system configuration

    3.  Finance Admin
        -   Access to payment logs, MoMo reports, and template purchases
        -   Export data for accounting and reconciliation

    4.  Content Admin
        -   Manage templates and categories
        -   Manage FAQ entries and static website copy

    -   Permissions should be role based.
    -   Super Admin can define which modules and actions each role can access.
    ```

  * **Admin Login Area**

    ```
    -   Admin login page:
        -   Email
        -   Password

    -   Security expectations:
        -   Account lockout after X failed attempts
        -   Session timeout after inactivity
        -   Forgot password flow (reset via email)
        -   Optional 2FA challenge after password
    ```

  * **Admin Dashboard (Home Screen)**

    ```
    Main metrics at a glance:

    -   Total registered users
    -   Total gifts sent
        -   Today
        -   This week
        -   This month
        -   All time
    -   Total MoMo volume
    -   Total service fee revenue
    -   Number of premium templates purchased
    -   Delivery status breakdown:
        -   Pending
        -   Delivered
        -   Read
        -   Failed
    -   Channel breakdown:
        -   App / Web
        -   USSD

    Quick links:

    -   View latest gifts
    -   View payment failures
    -   View error logs

    Graph widgets:

    -   Gifts sent per day
    -   MoMo volume per day
    -   Active users per day
    ```

  * **User Management Module**

    ```
    User List:

    -   Search by:
        -   Phone number
        -   Name
        -   Email

    -   Filters:
        -   Date registered (from / to)
        -   Status (Active / Deactivated)

    User Detail View:

    -   Profile information:
        -   Phone (main identity)
        -   Name
        -   Email (if any)
        -   Date created
        -   Last login

    -   Activity summary:
        -   Gifts sent (count + link to list)
        -   Gifts received (count + link to list)
        -   Premium templates purchased
        -   AI usage (number of generated messages)

    -   Admin actions:
        -   Deactivate or reactivate user
        -   Reset AI usage limit (future)
        -   View login history
        -   Impersonate user in read-only mode (see their portal view)
    ```

  * **Gift Management Module**

    ```
    Gift List:

    -   Search and filter gifts by:
        -   Gift code
        -   Sender phone
        -   Recipient phone
        -   Date range
        -   Status:
            -   Pending
            -   Delivered
            -   Read
            -   Failed

    Gift Detail View:

    -   Gift information:
        -   Gift code
        -   Sender details
        -   Recipient details
        -   Channel (Web, App, USSD)
        -   Template category and specific template
        -   Full message text
        -   Amount sent
        -   Service fee applied
        -   MoMo transaction IDs and network

    -   Timestamps:
        -   Gift created
        -   MoMo approved
        -   Notification sent
        -   Delivered
        -   Marked as read

    -   Notification log:
        -   SMS attempts
        -   WhatsApp attempts
        -   Email attempts
        -   Push notification attempts
        -   Status and provider response

    -   Admin actions:
        -   Resend notification (SMS / WhatsApp / Email)
        -   Manually mark as Delivered or Read (special support cases)
        -   Cancel or refund (if business rules allow, even if partial)
    ```

  * **Template Management Module**

    ```
    Template Categories:

    -   Manage high level categories:
        -   Gold
        -   Silver
        -   Blue
        -   Yellow
        -   Romantic
        -   Miscellaneous

    -   Actions:
        -   Add new category
        -   Rename category
        -   Disable category (hide from user side, but keep history)

    Template List:

    -   For each template show:
        -   Template name
        -   Category
        -   Type (Free / Premium)
        -   Price (for premium)
        -   Status (Active / Inactive)
        -   Preview thumbnail

    Template Detail:

    -   Template content:
        -   HTML / CSS for the card
        -   Mobile and desktop previews
        -   Short description
        -   Tags (Birthday, Apology, Thanks, Love, etc.)

    -   Flags and options:
        -   Free or Premium
        -   Default template for a category (used for USSD sends)

    -   Actions:
        -   Activate or deactivate
        -   Change price
        -   Assign as default for category
        -   Duplicate template to create variations
    ```

  * **Payment and Revenue Module**

    ```
    Transaction Logs:

    -   Record each payment:
        -   Transaction type:
            -   Gift send
            -   Template purchase
        -   Network:
            -   MTN
            -   Vodafone
            -   AirtelTigo
        -   Amount
        -   Service fee earned
        -   Status:
            -   Success
            -   Failed
        -   Reference IDs from MoMo / payment provider

    -   Filters:
        -   Date range
        -   Network
        -   Status
        -   Transaction type

    Revenue Summary:

    -   Total service fee revenue (selectable period)
    -   Total template sales (selectable period)
    -   MoMo failure rate

    Export:

    -   Export filtered transactions to CSV / Excel for accounting
    ```

  * **Notification Management Module**

    ```
    Notification Logs:

    -   For each notification store:
        -   Type:
            -   SMS
            -   WhatsApp
            -   Email
            -   Push
        -   Recipient identifier
        -   Related gift code
        -   Content preview (first 100 characters)
        -   Status:
            -   Queued
            -   Sent
            -   Delivered (if provider supports)
            -   Failed
        -   Provider response message
        -   Timestamp

    -   Filters:
        -   Type
        -   Status
        -   Date range

    Actions:

    -   Resend SMS / WhatsApp / Email for a specific gift
    -   Temporarily disable certain channels (for example, WhatsApp) if there is an outage
    ```

  * **USSD Monitoring Module**

    ```
    -   USSD session list:
        -   Phone number
        -   Menu path taken
        -   Result:
            -   Sent gift
            -   Viewed gift
            -   Abandoned

    -   USSD gifts summary:
        -   Number of gifts sent via USSD:
            -   Today
            -   This week
            -   This month
        -   Payment success vs payment failure

    -   Error logs for USSD sessions:
        -   Technical errors
        -   Timeouts
        -   Invalid input patterns

    -   Statistics:
        -   Average session duration
        -   Most common drop off step in the menu
    ```

  * **AI Message Generator Monitoring**

    ```
    Metrics:

    -   Total AI messages generated per day
    -   AI usage per user
    -   Cost estimation if using paid AI API

    Admin controls:

    -   Set AI usage limit per user per day (for example: 10 generations)
    -   Toggle AI feature globally (On / Off)
    -   Disable AI for specific abusive or suspicious users
    ```

  * **System Settings Module**

    ```
    Key configuration areas:

    1.  Payment Settings
        -   Service fee:
            -   Percentage
            -   Flat fee
        -   Template price limits
        -   MoMo API keys and endpoints (masked in UI)

    2.  Notification Settings
        -   Default channels:
            -   SMS required
            -   WhatsApp optional
            -   Email optional
        -   SMS gateway credentials
        -   WhatsApp API credentials
        -   Email SMTP settings

    3.  Security Settings
        -   Password policy for admin accounts
        -   Whether 2FA is required or optional
        -   Session timeout duration
        -   IP access rules (optional allow list)

    4.  Feature Flags
        -   Enable or disable:
            -   AI messaging
            -   Template purchases
            -   Specific MoMo networks (for maintenance or issues)
    ```

  * **Content Management (Optional but Useful)**

    ```
    -   Manage FAQ entries on the website
    -   Manage static content:
        -   "How It Works" text
        -   Footer link labels and descriptions
    -   Manage in-app or on-site announcements:
        -   Example: "New Valentine templates are now available"
    ```

  * **Support and Tickets Module (Lightweight)**

    ```
    -   Basic internal support tools:
        -   Log user complaints (created from support team)
        -   Link a ticket to:
            -   User
            -   Gift code

    -   Ticket status:
        -   Open
        -   In progress
        -   Resolved

    -   Internal support notes per ticket for context
    ```

  * **Audit Logs and Admin Activity**

    ```
    -   Log important admin actions:
        -   Login and logout
        -   Adding or editing templates
        -   Changing pricing or service fees
        -   Deactivating or deleting users
        -   Manual delivery or read status changes on gifts

    -   For each log entry store:
        -   Admin user
        -   Action performed
        -   Timestamp
        -   Before and after values (where possible)
    ```

  * **Technical Expectations for Admin Portal**

    ```
    -   It should consume secure backend APIs that enforce:
        -   Role based access control
        -   Authentication and authorisation

    -   UX expectations:
        -   Pagination and filtering on all large lists
        -   Strong input validation on all forms
    ```
